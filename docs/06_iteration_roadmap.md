# DeepCode V2 迭代路线图

> 版本：v2.0-roadmap  
> 周期建议：6 个迭代（每迭代 1-2 周）

---

## 一、总体节奏

| 迭代 | 目标 | 交付重点 |
|---|---|---|
| I0 | 基线修复 | 启动可靠性、命令可用性、诊断能力 |
| I1 | CLI 升级 | 会话管理、计划模式、任务流水线统一入口 |
| I2 | Web 控制台升级 | 任务中心、会话中心、工件中心 |
| I3 | 扩展与自动化 | MCP、Hook、Skill 初版 |
| I4 | 并行协作 | 多 Agent 团队、Worktree 并行 |
| I5 | 治理与企业化 | 策略审计、成本配额、组织级配置 |

---

## 二、各迭代详细计划

## I0：基线修复（P0）

### 目标

- 修复 deepcode 命令在本地不可用问题。
- 提供统一环境检查入口，避免“能安装但不能运行”。

### 任务

- 增加 deepcode doctor 命令（python、依赖、配置、端口检查）。
- 完善安装与开发环境说明（README）。
- 提供 UI/API 启动前置检查与友好报错。

### 验收

- 在新环境中 10 分钟内完成首次可运行。
- deepcode chat / run / serve / ui 都能给出明确成功或可执行失败提示。

### 风险与依赖

- 依赖环境差异（Windows/macOS/Linux）导致启动不一致。

---

## I1：CLI 升级（P0）

### 目标

- 从“聊天 CLI”升级为“工程 CLI”。

### 任务

- 新增 session 子命令：list/resume/rename/export/clear/fork。
- 新增 plan 命令：只读分析，生成可确认计划。
- 新增 task 子命令：list/get/stream/retry/cancel。
- 统一输出层：text + json。

### 验收

- 一个复杂需求可在 CLI 中完成：规划、执行、跟踪、恢复。
- 任务失败后支持 retry，且保留完整上下文。

### 风险与依赖

- CLI 状态管理与后端任务状态一致性。

---

## I2：Web 控制台升级（P0/P1）

### 目标

- WebUI 从演示页升级为运营控制台。

### 任务

- 新增 Dashboard（吞吐、成功率、耗时、失败 TopN）。
- 新增 Task Center（状态流 + 日志 + 重试）。
- 新增 Sessions（会话树 + 上下文占用）。
- 新增 Artifacts（Diff、测试报告、导出）。

### 验收

- 任务生命周期全链路可视化。
- 会话与工件可检索、可回放。

### 风险与依赖

- Streamlit 在复杂状态下的交互一致性与性能。

---

## I3：扩展与自动化（P1）

### 目标

- 引入生态扩展能力，降低硬编码成本。

### 任务

- MCP 管理：add/list/remove/auth（本地优先）。
- Hook 关键事件：PreToolUse、PostToolUse、TaskCompleted。
- Skill 目录机制：加载、禁用、参数注入。
- 安全门控：deny/ask/allow 规则与 UI 配置。

### 验收

- 至少接入 2 个 MCP 服务并稳定调用。
- 至少 3 类 Hook 触发与日志闭环可验证。

### 风险与依赖

- 第三方工具稳定性与权限边界。

---

## I4：并行协作（P1）

### 目标

- 支持多 Agent 并行协作与隔离执行。

### 任务

- Agent Team 视图（Planner/Coder/Reviewer/Tester）。
- Worktree 隔离执行与自动清理。
- Teammate 级状态与产出聚合。

### 验收

- 并行执行两个任务互不污染。
- 每个子 Agent 的日志与工件可单独追踪。

### 风险与依赖

- Git 工作树清理策略与跨平台路径问题。

---

## I5：治理与企业化（P2）

### 目标

- 提供组织级安全治理和运行治理能力。

### 任务

- 托管策略源与优先级（Managed > Local > Project > User）。
- 审计日志导出（操作、审批、策略命中）。
- 模型成本统计与预算告警。
- 配置快照与回滚。

### 验收

- 关键操作可审计、可追责、可回放。
- 配额超限时有明确降级与提示机制。

### 风险与依赖

- 策略复杂度提升后易造成误拦截。

---

## 三、里程碑验收（跨迭代）

- M1：命令入口与环境可靠性达标（I0）
- M2：CLI 工程化闭环可用（I1）
- M3：Web 控制台可运营（I2）
- M4：扩展生态可插拔（I3）
- M5：并行协作稳定（I4）
- M6：治理能力可上线（I5）
