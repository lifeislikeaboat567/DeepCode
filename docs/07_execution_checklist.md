# DeepCode V2 执行清单

> 目标：把路线图转成可直接执行的工程任务单（按周推进）

---

## 一、执行规则

1. 每个迭代只做承诺范围内的事项，避免 scope 膨胀。  
2. 每个功能点必须有验收脚本或验收步骤。  
3. 新功能默认要求：日志、错误码、用户提示三件套。  
4. 所有高风险工具调用必须经过策略检查。

---

## 二、I0 执行清单（立即开始）

### 2.1 开工前

- 检查开发环境（Python 版本、依赖、可执行路径）。
- 统一启动方式（本地、容器、CI）。
- 建立问题看板：安装失败、启动失败、运行失败三类。

### 2.2 开发项

- 增加 doctor 命令并输出结构化检查项。
- 启动命令增加 preflight 检查。
- README 增加“最短可运行路径”。

### 2.3 验收项

- 新机器按文档 10 分钟内跑通。
- 所有启动失败都能给出可执行修复建议。

---

## 三、I1 执行清单（CLI 工程化）

### 3.1 开发项

- 实现 session 子命令族。
- 实现 plan 命令与可确认执行。
- 实现 task 子命令族与流式状态。
- 增加 json 输出模式。

### 3.2 测试项

- 单元测试：命令参数与状态迁移。
- 集成测试：从计划到执行到重试闭环。

### 3.3 验收项

- 复杂任务可完整走通且可恢复。

---

## 四、I2 执行清单（Web 控制台）

### 4.1 开发项

- Dashboard：关键指标与活动流。
- Task Center：步骤流 + 日志 + 重试。
- Sessions：会话树 + 上下文占用。
- Artifacts：Diff + 报告 + 导出。

### 4.2 交互项

- 统一状态色与反馈机制（pending/running/success/failed）。
- 所有异步动作提供 loading 与失败反馈。

### 4.3 验收项

- UI 可独立完成任务监控与失败定位。

---

## 五、I3 执行清单（扩展与自动化）

### 5.1 开发项

- MCP 配置与运行命令。
- Hook 引擎关键事件接入。
- Skill 加载器与开关。
- 策略规则编辑与检查器。

### 5.2 验收项

- 接入 2 个以上外部集成，任务可稳定调用。

---

## 六、I4 执行清单（并行协作）

### 6.1 开发项

- 多 Agent 任务分工与状态聚合。
- Worktree 生命周期管理。
- 并行任务冲突检测与提示。

### 6.2 验收项

- 并行任务互不污染，产物可追溯到 Agent。

---

## 七、I5 执行清单（治理与企业化）

### 7.1 开发项

- 托管策略优先级实现。
- 审计日志导出和检索。
- 成本预算与告警阈值。

### 7.2 验收项

- 策略与审计能力满足上线前合规检查。

---

## 八、每周节奏模板

1. 周一：冻结本周范围与验收项。  
2. 周二到周四：开发 + 单测 + 集测。  
3. 周五：演示、回归、文档更新、风险复盘。

---
