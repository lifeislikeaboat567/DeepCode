# DeepCode 24h Personal Agent 迭代计划（V1）

## 目标
将 DeepCode 从“可对话工具”升级为“24h 在线、可主动推进、可持续记忆、可审慎执行”的个人 Agent。

## 现状观察（基于代码）
- 已有基础闭环：normalize -> plan -> execute -> validate -> reflect。
- 已有策略引擎与审批存储能力，但默认规则为空时会走 allow。
- 会话可持久化，但缺少自动上下文压缩与任务记忆库统一入口。
- Agent 聊天有技能/MCP 上下文注入，但缺少任务记忆 RAG 注入。

## 本轮已立即执行
1. 身份提示词外置
- 新增仓库根目录 `SOULS.MD`，定义 DeepCode 的身份、主动性、安全边界与记忆准则。
- Prompt 层统一从 `SOULS.MD` 加载身份指令（缺失时回退默认身份）。

2. 自动上下文压缩
- 新增会话压缩机制：当上下文估算 token 超阈值，自动将旧消息压缩为 system 摘要并保留最近消息。
- 压缩记录写入 session metadata 的 `context_summaries`，用于追踪压缩历史。
- 压缩在发送与重发消息流程中自动触发。

3. 任务记忆库（本地持久化 + 可选向量检索）
- 新增 `TaskMemoryStore`：以 JSON 持久化任务请求/结果/过程。
- 同步写入向量内存（Chroma 可用时生效），支持 lexical + vector 融合检索。
- 聊天 Agent 快照与任务中心执行结果都会写入任务记忆库。

4. 会话删除联动记忆删除
- 删除单个会话/分组会话时，同步删除记忆库中对应 `session_id` 的条目。

5. RAG 注入（任务记忆）
- Ask 模式：系统上下文注入“Task memory context”。
- Agent 模式：将记忆命中注入 Agent runtime context 与任务提示词。

## 下一阶段（建议优先级）
P0
- 审批闭环前端化：在 UI 展示 pending approval，支持一键 approve/reject 后自动续跑。
- 默认安全策略模板：首次启动自动写入高风险动作 `ask` 规则（可关闭）。

P1
- 24h 在线守护进程：
  - 心跳与健康检查
  - 自动重启与失败告警
  - 定时任务（夜间索引、记忆整理）
- 记忆质量治理：
  - 去重与衰减
  - 成功案例优先级提升
  - 失败案例带风险标签

P2
- 主动执行增强：
  - 缺条件自创建（测试数据、脚本、临时环境）策略模板化
  - “遇阻自动切换方案”策略树
  - 多轮任务自动拆分与阶段验收

## 验收标准（V1）
- 长对话下可自动压缩并继续完成任务。
- 任务完成后可被记忆检索命中并用于后续回答。
- 删除对话后，不再检索到该会话关联记忆。
- 身份提示可通过 `SOULS.MD` 独立维护。
